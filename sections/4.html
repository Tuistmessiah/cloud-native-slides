🐳 What is Docker?

Contrary to a virtual machine, a container does not have its own operating system. It shares the host's OS.

⬇️ Why Use Containers?

🔹 Why Use Containers?

  • 📦 Portable across machines and environments
  • ⚡ Fast startup time and small footprint
  • 🔁 Reproducible builds and behavior
  • 🚫 Avoids "works on my machine" problems

⬇️ Docker Image vs Container

🧊 Image vs 🟢 Container

  • Image: A snapshot of your app and dependencies
  • Container: A running instance of an image
  • You can run many containers from the same image

⬇️ Hands-On: Hello from Docker

🔧 Hands-On: Hello from Docker

Setup docker in your machine

docker pull alpine
docker run -it alpine
# Try inside the container:
echo hello docker
exit

This spins up a tiny Linux container interactively.

⬇️ See What You Did

🔍 Inspect What You Did

  • 📜 docker ps -a – See all containers (even stopped ones)
  • 🧼 docker rm <id> – Remove a container
  • 🧯 docker rmi <image> – Remove the image

⬇️ Summary

✅ Summary
